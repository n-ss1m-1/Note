DQL：对表的数据进行查询



一：基本查询

 	1.查询多个字段：**select**  字段1，字段2，...   **from**  表名;

 	  (查询所有字段)  select   **\***   from  表名;

 	2.设置别名： select 字段1\[**as** 别名1]，字段2\[as 别名2],...  from 表名；

 			       注：AS可以省略

 	3.去除重复记录：select  **distinct**  字段1，字段2，...  from  表名



二：条件查询

 	1.语法： select  字段列表  from  表名  **where  条件列表**；

 	2.条件：某字段 + 条件

 		      	  >     >=      <     <=     =     <>    !=

 			  闭区间between...and...(1填小 2填大)

 			  匹配列表 in(...) : 相当于多个or  例：age in(20,30,40)

 			  like 占位符(单个\_    任意个%)  (前面不需要=)

 			  is null    is not null   非匹配列表 not in()

 		  	  and(\&\&)      or(||)       not(!)



三：聚合函数

 	1.定义：将一列数据作为整体，进行纵向计算。

 	2. 常用：

 		count：统计数量					

&nbsp;			1.count(distinct 字段)  统计不重复的个数     

&nbsp;			2.group by  分组统计

 		max：最大值

 		min：最小值

 		avg：平均值

 		sum：求和

 	3.语法： select    **聚合函数(字段)**    from   表名;

 	4.注意：1.null值不参与计算   2.聚合函数(distinct+字段)   只计算该字段不重复的行





四:分组查询（分组:一组一行）

 	1.语法：select  字段列表  from  表名  \[where  条件]   **group  by**  分组字段名  \[having  条件]

 	  2.区分where和having:

 		1)执行时机不同：where是分组前过滤，having是分组后过滤。

 	        2)▲判断条件不同：where不能对聚合函数的结果进行判断，而having可以。

 	 3.**注意**：1）分组后使用聚合函数统计。然后再用having进行过滤。

 		      2）分组后查询的字段一般为**分组字段和聚合函数**，其他则没有意义。

 



五：排序查询

 	1.语法：select  字段列表  from  表名  **order by**  字段1  排序方式1，字段2  排序方式2...;

 	  2.排序方式： 1）ASC：升序(默认可省略)

 				2）DESC：降序

 	 3.注意：如果是多字段排序，则当字段1值相同时，才根据字段2进行排序。

 



六：分页查询

 	1.语法：select  字段列表  from  表名  **limit  起始索引，查询记录数**；

 	2.注意：

 			 1）起始索引从0开始，起始索引 = (查询页数-1) \* 每页显示记录数(查询页数从1开始算起)

 		         2）分页查询是数据库的方言，不同数据库有不同的实现。

 		         3）如果查询的是第一页数据，则起始索引可以省略。





七：编写顺序和执行顺序：

 	编写顺序：select  字段列表  		执行顺序：from 表名列表

 			from  表名列表					where  条件列表

 			where  条件列表  				group by 分组字段列表

 			group by  分组字段列表   			having  分组后条件列表

 			having  分组后条件列表 			select  字段列表

 			order by  排序字段列表  			order by  排序字段列表

 			limit  分页参数;					limit  分页参数
