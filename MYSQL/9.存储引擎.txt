存储引擎

MySQL体系结构：
1.连接层
负责客户端与 MySQL 服务器的连接管理、用户身份验证、权限校验及安全控制，确保客户端安全接入并执行其权限范围内的操作。
2.服务层
是 MySQL 的核心服务层，提供 SQL 接口、查询缓存、SQL 解析与优化、内置函数执行等功能，同时实现所有跨存储引擎的通用操作（如存储过程、自定义函数）。
3.引擎层
负责数据的实际存储与读取逻辑，通过 API 与服务层通信。支持多种存储引擎（如 InnoDB、MyISAM），可根据业务需求选择不同特性的引擎。索引在此层实现->不同引擎的索引不同
4.存储层
位于最底层，将数据持久化到文件系统中，负责与存储引擎交互，完成数据文件、日志文件等的物理存储管理。
==========================
存储引擎基础操作
1.指定引擎建表：创建表时通过 ENGINE = 引擎名（如 ENGINE = INNODB）显式指定表的存储引擎。
2.查看支持引擎： SHOW ENGINES; 
==========================
InnoDB 定位与特性
定位：兼顾高可靠性与高性能的通用存储引擎，MySQL 5.5 及以上版本的默认存储引擎。
特性：
1.事务支持：DML(增删改) 操作遵循 ACID 模型，保障【事务】的原子性、一致性、隔离性和持久性。
2.并发优化：采用【行级锁】，提升多线程并发访问的性能，减少锁冲突。
3.数据完整性：支持【外键】约束，强制表间关联的合法性，保证数据准确性。
//三、InnoDB 存储相关
物理文件：开启 innodb_file_per_table 参数后，每张 InnoDB 表对应一个独立的表空间文件（xxx.ibd），存储表结构（配合 .frm/.sdi 文件）、数据及索引。
逻辑存储结构：采用分层级的存储架构，从大到小依次为：表空间（Tablespace）→ 段（Segment）→ 区（Extent，1M）→ 页（Page，16K）→ 行（Row），层级关联构成完整的数据存储体系。
==========================
//MyISAM 存储引擎
定位：MySQL 早期的默认存储引擎。
核心特点：
1.不支持事务与外键
2.仅支持表锁，无行锁
3.数据访问速度快
存储文件：
xxx.sdi：表结构信息
xxx.MYD：数据文件
xxx.MYI：索引文件
==========================
//Memory 存储引擎
定位：数据存储在内存中，仅适合作为临时表或缓存。
核心特点：
1.数据全存于内存（断电 / 故障会丢失）
2.默认使用 Hash 索引
3.存储文件：仅 xxx.sdi 保存表结构，数据 / 索引均在内存。
==========================
对比：   InnoDB	MyISAM
事务：     支持		 不支持
锁机制：  行锁		  表锁
外键	       支持		 不支持
