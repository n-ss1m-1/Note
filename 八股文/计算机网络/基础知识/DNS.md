# DNS是什么
**DNS**(Domain Name System)是一种将域名转换为IP地址的**分布式系统**。(域名易于记忆，IP不易记忆)

# 分布式设计的原因(集中式设计的问题)
1. 单点故障 -> 网络瘫痪
2. 通信容量巨大 -> 一台服务器难以满足
3. 远距离查询 -> 严重时延
4. 维护成本大

# 域名的层级关系
分隔符：句点( . )
级别：越靠右的位置，层级越高(越靠近根)

# DNS解析过程(查到即返回)
1. 查询**浏览器缓存**
2. 查询**本地计算机缓存**(Host文件)
3. 主机向**本地DNS服务器**发送DNS查询请求
4. 本地DNS服务器查看自身是否**缓存**该域名的IP地址
5. 本地DNS服务器向**根DNS服务器**发送查询请求，根DNS服务器不负责解析域名，但是会告诉本地DNS服务器应该前往哪个顶级域名DNS服务器查询下一步的信息
6. 本地DNS服务器接着向指定的**顶级域名DNS服务器**发送查询请求，该顶级域名DNS服务器不负责解析域名，但是会告诉本地DNS服务器应该前往哪个权威DNS服务器查询下一步的信息
7. 本地DNS服务器最后向指定的**权威DNS服务器**发送查询请求，权威DNS服务器返回查询结果给本地DNS服务器
8. 本地DNS服务器将收到的IP地址返回给浏览器，并将域名解析结果缓存在本地
9. 浏览器使用IP地址发起连接，获取网页内容

# 递归查询与迭代查询
1. 递归查询
- DNS客户端只需发送一个查询请求，然后等待完整的解析结果。
- 上层DNS服务器会自行查询下一级的服务器，并将最终结果返回给客户端。
2. 迭代查询
- DNS客户端只询问上层DNS服务器A一个更底层的DNS服务器B的地址，然后再自行向DNS服务器B发起查询请求，以此类推，直到获取完整的解析结果为止。
1. 递归查询用于客户端(用户)，迭代查询用于DNS服务器之间的通信。
