#### 283. 移动零

##### 怎么想到用双指针？

题目要求**原地操作 + 保持非零元素相对顺序**，用「快慢指针」分工最清晰：

- 快指针（r）：遍历数组，专门找**非零元素**；
- 慢指针（l）：记录**下一个非零元素该放的位置**。
    
    避免了额外数组空间，时间复杂度 O (n)。

##### 具体怎么用？

1. `l=0, r=0`，r 遍历数组：
    
    - 若`nums[r]!=0`，则`nums[l]=nums[r]`，`l++`；
    
2. 遍历结束后，从`l`到数组末尾统一填 0。

##### 易错点

- 填零必须从`l`开始，别从 0 开始覆盖已填好的非零元素；
- 非零元素赋值后`l`要及时 ++，别漏。

##### 边界判断

- r 遍历范围：`0 ≤ r < n`；
- 填零范围：`l ≤ r < n`。

#### 11. 盛最多水的容器

##### 怎么想到用双指针？

暴力枚举所有两条线的组合是 O (n²)，面试会超时。

核心观察：**容器面积 = (右 - 左) × min (左高度，右高度)**。

- 若移动**较高的边**：宽度减小，且高度仍由原来的短边决定（不会变大），面积**必然减小**；
- 若移动**较矮的边**：宽度减小，但有可能找到更高的边，让高度变大，面积**有可能增大**。
    
    因此用**双指针从两端向中间贪心收缩**，直接把时间降到 O (n)。

##### 具体怎么用？

1. 初始化：`l=0`（左端点），`r=height.size()-1`（右端点）；
2. 先算**初始面积**：`ans = (r-l) * min(height[l], height[r])`；
3. 循环（`l < r`）：
    
    - 若`height[l] < height[r]`：移动左指针（`l++`），争取找到更高的左边界；
    - 否则：移动右指针（`r--`），争取找到更高的右边界；
    - 每次移动后**重新计算面积**，更新`ans`。
    

##### 易错点

1. **初始面积必须先算**：别直接进循环移动指针，漏掉两端的最大面积情况；
2. **移动指针后再算面积**：顺序别搞反（先移动，再计算新的 l 和 r 对应的面积）；
3. 高度是取`min`不是`max`：别记错面积公式。

##### 边界判断

- 循环条件严格`l < r`：当`l == r`时停止，避免重复计算单条线的情况；
- 移动指针后仍需保证`l < r`：不用额外判断，循环条件会兜底。

#### 15. 三数之和

##### 怎么想到用双指针？

1. 暴力枚举三元组是 O (n³)，完全不可行；
2. **排序**是关键：
    
    - 排序后可以**固定第一个数`i`**，剩下两个数`j`和`k`用**双指针从两端向中间找**，把 O (n²) 的两重循环优化为 O (n) 的双指针遍历；
    - 排序后方便**跳过重复元素**，避免输出重复的三元组。
    

##### 具体怎么用？

1. **先排序数组**（核心前提）；
2. 枚举第一个数`i`（范围：`0 ≤ i ≤ n-3`，给`j`和`k`留位置）：
    
    - **去重 i**：若`i>0`且`nums[i] == nums[i-1]`，直接`continue`（避免重复的三元组起点）；
    - **小优化 1**：若`nums[i] + nums[i+1] + nums[i+2] > 0`，直接`break`（后续 i 更大，三个数的和只会更大，不可能等于 0）；
    - **小优化 2**：若`nums[i] + nums[n-2] + nums[n-1] < 0`，直接`continue`（当前 i 的所有组合都太小，快进到下一个 i）；
    
3. 初始化双指针：`j = i+1`（左端点），`k = n-1`（右端点）；
4. 循环（`j < k`）：
    
    - 计算`sum = nums[i] + nums[j] + nums[k]`；
    - 若`sum < 0`：需要更大的数，`j++`；
    - 若`sum > 0`：需要更小的数，`k--`；
    - 若`sum == 0`：
        
        - 记录三元组：`ans.push_back({nums[i], nums[j], nums[k]})`；
        - **去重 j**：`j++`，同时跳过所有`nums[j] == nums[j-1]`的情况（保证`j < k`）；
        - **去重 k**：`k--`，同时跳过所有`nums[k] == nums[k+1]`的情况（保证`j < k`）。
        
    

##### 易错点（面试高频坑）

1. **必须先排序**！否则既无法用双指针，也无法去重；
2. **i 的范围是`i < n-2`**：不然`j=i+1`和`k=n-1`会越界；
3. **去重 i 时必须`i>0`才比较**：否则`i=0`时访问`nums[i-1]`会越界；
4. **sum= =0 后，j 和 k 都要去重**：只去重一个会导致重复三元组；
5. **去重 j 和 k 时必须保证`j < k`**：别跳过头导致`j > k`。

##### 边界判断

- 外层循环：`i < n-2`；
- 内层循环：`j < k`；
- 去重 j：`j++`后仍需`j < k`且`nums[j] == nums[j-1]`才继续跳；
- 去重 k：`k--`后仍需`j < k`且`nums[k] == nums[k+1]`才继续跳。

#### 42. 接雨水

##### 怎么想到用双指针？

核心公式：**位置 i 的雨水量 = min (左边最大高度 lmax, 右边最大高度 rmax) - height [i]**。

- 若用两个数组分别存每个位置的`lmax`和`rmax`，需要 O (n) 空间；
- 双指针可以**动态维护`lmax`和`rmax`**，在遍历过程中直接计算雨水量，把空间降到 O (1)。
    
    关键贪心逻辑：
    
    - 若`lmax < rmax`：说明当前左指针位置的雨水量**由`lmax`决定**（因为右边有更高的`rmax`兜底），可以直接计算左指针位置的雨水量，然后移动左指针并更新`lmax`；
    - 反之：计算右指针位置的雨水量，移动右指针并更新`rmax`。
    

##### 具体怎么用？

1. 初始化：
    
    - `l=0`（左端点），`r=height.size()-1`（右端点）；
    - `lmax=height[0]`（左边当前最大高度），`rmax=height[r]`（右边当前最大高度）；
    - `ans=0`（总雨水量）；
    
2. 循环（`l < r`）：
    
    - 若`lmax < rmax`：
        
        - `ans += lmax - height[l]`（计算左指针位置的雨水量）；
        - `l++`（移动左指针）；
        - `lmax = max(lmax, height[l])`（更新左边最大高度）；
        
    - 否则：
        
        - `ans += rmax - height[r]`（计算右指针位置的雨水量）；
        - `r--`（移动右指针）；
        - `rmax = max(rmax, height[r])`（更新右边最大高度）。
        
    

##### 易错点（困难题高频坑）

1. **初始`lmax`和`rmax`要设为`height[0]`和`height[r]`**！别设为 0，否则如果第一个柱子很高，会算错初始雨水量；
2. **计算顺序不能乱**：先算`ans` → 再移动指针 → 最后更新`max`！
    
    - 比如左指针位置的雨水量是用 ** 当前的`lmax`** 算的，算完后移动`l`，再看新的`l`是不是更高，更新`lmax`；
    
3. **不用额外判断`height[l]/height[r] > max`**：此时`max - height[i] ≤ 0`，`ans`加 0 没问题，不影响结果；
4. **别搞反`lmax`和`rmax`的比较**：`lmax`小就处理左指针，`rmax`小就处理右指针。

##### 边界判断

- 循环条件严格`l < r`：当`l == r`时停止，最后一个位置的雨水量已经被处理过（或者说它自己不会存水）；
- 更新`lmax`和`rmax`时，必须取`max(当前max, 新的height[l]/height[r])`：保证`lmax`和`rmax`是**单调递增**的，不会变小。
