#### 前置知识：中断、异常、系统调用的区别
##### 1. 中断（Interrupt / 硬件中断）

中断是**异步**发生的，通常由**外部硬件设备**或**定时器**触发。
- **目的**：通知CPU有“外部事件”需要处理，实现CPU与外设的并行工作。
- **来源**：来自CPU外部。例如：
    - **I/O设备**：键盘敲击、鼠标移动、硬盘读写完成。
    - **定时器**：操作系统用来实现时间片轮转、超时检测。
- **处理**：
    - 中断处理程序通常执行完后会返回到被中断的指令**下一条**继续执行。
    - CPU可以在指令执行的间隙响应中断。

##### 2. 异常（Exception / 执行错误）

异常是**同步**发生的，通常由CPU在**执行指令时**检测到的错误或特殊条件触发。
- **目的**：处理程序错误、实现特权操作（如系统调用）或管理虚拟内存。
- **来源**：来自CPU内部，与当前正在执行的指令直接相关。例如：
    - **故障(Fault)**：缺页异常（Page Fault）、除零错误。
    - **陷阱(Trap)**：**系统调用**（System Call）。
    - **中止(Abort)**：非法指令、硬件错误。
- **处理**：
    - **故障**：处理完后通常会**重新执行**触发异常的那条指令（如缺页异常加载页面后重试）。
    - **陷阱**：执行完后返回到下一条指令。
    - **中止**：通常导致程序终止。
##### 3. 系统调用属于陷阱类异常



### I/O设备管理
#### 设备控制器
- **作用**：作为**CPU和物理设备**之间的翻译官，屏蔽硬件细节，向 CPU 提供标准化寄存器接口
- **三类核心寄存器**：
    - **命令寄存器**：CPU 向设备下发指令（如「读取数据」「开启设备」）
    - **状态寄存器**：设备向 CPU 反馈当前状态（如「忙」「就绪」「完成」）
    - **数据寄存器**：暂存待传输的数据，减少 CPU 与设备的直接交互
    - **数据缓冲区**：用于批量传输数据(块设备)
- **设备分类**：
    - **块设备**：按固定大小块读写（如硬盘、U 盘），可随机寻址
    - **字符设备**：按字节流读写（如键盘、鼠标），不可寻址

#### I/O 控制方式
1. **轮询**（Polling）
- 原理：CPU 不断查询设备状态寄存器，直到 I/O 完成
- 缺点：**CPU 全程阻塞**，利用率极低，仅适合简单设备

1. **[[1.硬件结构#软中断|中断]]**（Interrupt）
- 原理：设备完成 I/O 后**主动**向 CPU 发中断，CPU 暂停当前任务，处理 I/O 结果
- 优点：解放 CPU，适合频繁小数据 I/O（如键盘、鼠标）
- 缺点：频繁中断会占用大量 CPU 时间，不适合磁盘等大数据设备

1. **DMA**（直接内存访问）
- 原理：让 **DMA 控制器直接在设备与内存之间传输数据**，**CPU 仅需发起指令和接收完成中断**
- 流程：
    1. CPU 配置 DMA（传输大小、目标内存地址）
    2. DMA 控制器指挥设备传输数据到内存
    3. 传输完成后 DMA 发中断通知 CPU
- 优点：**彻底解放 CPU**，适合磁盘、网卡等大数据量 I/O

#### 设备驱动程序
- **作用**：
	- **操作系统与硬件**之间的“桥梁”
	- **屏蔽不同设备控制器的差异** ↓
	- 向操作系统内核提供**统一接口**（如 `open`/`read`/`write`/`ioctl`）
- **核心职责**：
    - 注册设备的**中断处理程序**
    - 操作设备控制器寄存器，执行 I/O 操作
    - 处理设备中断，完成数据读写与格式转换

#### 通用块层
- **作用**：**块设备专属**为所有块设备提供统一抽象，优化 I/O 性能
- **核心功能**：
    1. **I/O 调度**：对 I/O 请求排队、重排序、合并，减少磁盘寻道时间
    2. **提供标准接口**：向上对接文件系统，向下对接不同设备驱动
- **Linux 5 种 I/O 调度算法**：
    - 无调度算法：不做任何处理，交给物理设备（虚拟机场景）
    - 先入先出（FIFO）：按顺序处理请求（简单场景）
    - 完全公平调度（CFQ）：每个进程分配时间片，保证公平（默认）
    - 优先级调度：高优先级请求先执行（桌面 / 多媒体）
    - 最终期限调度（Deadline）：设置截止时间，避免饥饿（数据库 / 高 I/O 压力场景）

#### 存储系统 I/O 软件分层(Linux)
![[存储系统IO软件分层.PNG|256]]
- **文件系统层**：包括虚拟文件系统和其他文件系统的具体实现，它向上为应用程序统一提供了标准的文件访问接口，向下会通过通用块层来存储和管理磁盘数据。
- **通用块层**：包括块设备的 I/O 队列和 I/O 调度器，它会对文件系统的 I/O 请求进行排队，再通过 I/O 调度器，选择一个 I/O 发给下一层的设备层。
- **设备层**：包括硬件设备、设备控制器和驱动程序，负责最终物理设备的 I/O 操作。


#### 键盘输入 `A` 的完整过程

| 步骤               | 详细描述                                             |
| ---------------- | ------------------------------------------------ |
| **1. 物理按键**      | 用户按下键盘上的“A”键，机械触点闭合，产生电信号                        |
| **2. 键盘控制器响应**   | 控制器检测到按键事件，生成对应的**扫描码**（如 0x1E）                  |
| **3. 发送中断**      | 键盘控制器通过总线向 CPU 发送中断请求（IRQ）                       |
| **4. CPU 响应中断**  | 保存当前进程上下文，跳转到中断处理程序                              |
| **5. 中断处理程序执行**  | 调用键盘驱动程序的中断处理函数                                  |
| **6. 读取扫描码**     | 驱动程序从键盘控制器的数据寄存器读取扫描码                            |
| **7. 解码为 ASCII** | 根据扫描码表，将 0x1E 映射为 ASCII 字符 `'A'`（0x41）           |
| **8. 放入读缓冲区**    | 将 `'A'` 写入终端设备的读缓冲区（如 `/dev/tty`）                |
| **9. 用户程序读取**    | 如果你在运行 `cat` 或 `echo`，它会调用 `read()` 从缓冲区读取 `'A'` |
| **10. 显示输出**     | 终端程序将 `'A'` 发送到显示器驱动，驱动将其写入显存，最终显示在屏幕上           |
