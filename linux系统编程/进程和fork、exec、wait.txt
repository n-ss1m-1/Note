进程和fork、exec、wait
进程：
	程序：死的，只占用磁盘空间
	进程：活的，是运行起来的程序，占用内存、CPU等资源


▲fork函数：
	1.函数原型： pid_t fork(void)
	2.创建子进程，父子进程各自返回：父进程返回子进程pid，子进程返回得到0
	3.getpid()获取当前进程的pid    getppid()获取当前进程的父进程的pid
	4.循环创建n个子进程模型：


父子进程相同的属性：
	data段、text段、栈、堆、环境变量、全局变量、宿主目录位置、进程工作目录位置、信号处理方式
▲父子进程不同的属性：
	进程id、返回值、各自的父进程、进程创建时间、闹钟、未决信号集
▲父子进程共享的属性：
	1.全局变量：读时共享，写时复制
	2.文件描述符、mmap映射区


▲exec函数族：
	1.作用：使进程执行某一程序。	
	2.返回值：成功无返回值，失败返回-1
	3.函数原型：
		1）int execlp(const char * file,const char * arg, ...)   一般用于内置命令
			参数1：程序名
			参数2：argv[0] 程序名
			参数3：argv[1] 传入的参数1
			...
			必须以NULL结尾
		2）int execl(const char * path,const char * arg, ...) 一般用于自定义的程序
		3）int execvp()
		说明：
			p：会根据PATH环境变量寻找参数
			l：逐个填入arg，且必须以NULL作为最后一个参数
			v：可以以数组形式填入参数


孤儿进程：父进程先于子进程终止，子进程沦为“孤儿进程”，会被init进程领养。
僵尸进程：子进程终止，父进程尚未对子进程进行回收，在此期间，子进程沦为“僵尸进程”。kill对其无效。


▲wait函数：
	1.函数原型：pid_t wait(int * status)
	2.参数：(传出的)回收的子进程的状态
	3.返回值：
		成功：回收的子进程的pid
		失败：-1   errno	-> 当没有可回收的子进程时，wait() 会返回 -1，且 errno 被设为 ECHILD
	4.函数作用：
		1）阻塞等待子进程退出(当还有子进程在运行时)
		2）清理子进程残留在内核的pcb资源
		3）通过传出status，得到子进程结束状态
	5.获取子进程正常终止值：
		WIFEXITED(status) -> 为真 -> 调用WEXITSTATUS(status) -> 得到子进程的退出值
	6.获取导致子进程异常终止的信号：
		WIFSIGNALED(status) -> 为真 -> 调用WTERMSIG(status) -> 得到导致子进程异常终止的信号编号


▲waitpid函数：
	1.函数原型：pid_t waitpid(pid_t pid, int * status, int options)
	2.参数：
		1）pid：指定回收的某一子进程的pid
			>0：待回收的子进程pid
			=0：同组的任一子进程
			=-1：任意一个子进程
			<-1：其绝对值所表示进程组内的任一进程
		2）status：(传出的)回收的子进程的状态
		3）options：WNOHANG 指定回收方式为 非阻塞(无需求可传0)
	3.返回值：
		>0：表示成功回收的子进程pid
		=0：函数调用时使用了WNOHANG 且尚未回收到子进程
		-1：失败  errno

注意：wait和waitpid 都是 一次调用=回收一个子进程
	  若想回收多个，则需要调用多次。
